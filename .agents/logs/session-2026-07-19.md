# Session 2026-07-19

## Headless tests broke the host Flatpak document portal

### Symptom

After an Anvil test run, Flatpak applications failed during Bubblewrap setup because their
`/run/user/$UID/doc/by-app/<application-id>` source paths no longer existed. Restarting the desktop
portal restored the document mount, but a later test run could break it again.

### Root cause

The shared test launcher started its private D-Bus daemon with `--fork`. The tracked process exited
immediately while the real daemon and D-Bus-activated portal services survived teardown. That
private bus also inherited the host `XDG_RUNTIME_DIR` and display variables, so its private
document portal operated on the host `/run/user/$UID/doc` mount.

`test_host_guard.py` did not directly change the desktop environment, but the guard omitted
`XDG_RUNTIME_DIR` and therefore could not reject this unsafe session layout.

### Fix

The D-Bus launcher now uses `--nofork`, returns the live daemon process, and removes inherited host
bus and display variables. Both headless session paths create a private mode-0700 runtime directory,
publish it to the private D-Bus activation environment, and wait for the owned daemon during
teardown. Host-safety checks now validate the runtime path alongside the data, config, and cache
roots.

### Validation and recovery

- Focused runner, host-guard, and shell-session tests passed, including live checks that the exact
  D-Bus daemon PID exits after each session.
- The icon-tag E2E suite passed 2/2 in `fedora-devbox`, with its Wayland socket under the private
  runtime and no new D-Bus or portal processes remaining afterward.
- The full `npm test` gate passed: 55 tiling tests, 1,097 unit tests, and 50 tooling tests with two
  host-smoke tests skipped as expected.
- Both opt-in live host-smoke tests passed, and the relevant process list was unchanged across each
  run.
- The leaked test-owned buses were terminated, the host document and desktop portals were
  restarted, and `flatpak run --command=true org.inkscape.Inkscape` succeeded.
