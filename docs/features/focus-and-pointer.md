# Focus and pointer

Anvil can emphasize keyboard focus visually, move the pointer with keyboard focus, or make focus follow the pointer.

## Focus border

A colored border around the active window is enabled by default. Toggle it with <kbd>Super</kbd>+<kbd>X</kbd> or **Anvil Preferences → Appearance → Border around focused window**.

The optional **Window split hint border** shows the active container's split direction. Border colors, widths, corner radius, and shadows are configured on the Appearance page.

## Move pointer with focused window

Enable **Anvil Preferences → Tiling → Move pointer with focused window** to warp the pointer into the tile selected by keyboard focus or swapping. When focus returns to a previously used window, Anvil keeps track of the pointer's last useful position where possible.

## Focus on hover

Enable **Focus on Hover** to focus and raise the normal window underneath the pointer. Anvil pauses hover changes during workspace transitions, while the overview is visible, and while a dialog owns focus.

Both pointer options are disabled by default and currently marked experimental in preferences.
